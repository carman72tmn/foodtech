# CRM-модули Result.Rest

> Источник: https://result.gitbook.io/result.rest-dokumentaciya/dokumentaciya

## Компания

Общие настройки компании.

## Филиалы

### Создание филиала

- **Название** — адрес или любое характеризующее название
- **GMT** — часовой пояс (влияет на время работы, рассылки)
- **Слайды в шапке** — баннеры для ПК и смартфон
- **Контакты** — телефоны, Telegram-ссылка
- **Оформление заказа**: скрытие стоимости доставки, время предзаказа, поля формы, способы оплаты, точки самовывоза
- **Внешние программы учета**: уведомления об ошибках (e-mail, VK, Telegram), интеграция с CRM (IIKO / FrontPad / Нет)
- **Платежные системы**, **Метрики**, **Сервисы рассылок**, **SMS сервисы**
- **Шаблоны сообщений**

## Каталог

### Подразделы:

- **Разделы** — категории меню
- **Товары** — позиции меню с описанием, ценами, размерами
- **Опции** — модификаторы к товарам

## Заказы

- Список всех оформленных заказов
- Поиск по ID / Имя / Телефон / Дата
- Поля: ID, Создан, Клиент, Сумма, Скидка, Итого, Оплата, Источник, Статус, Оценка
- **Отключение приёма**: чекбоксы для доставки / самовывоза / обоих
- **Экспорт заказов**: формат, кодировка, e-mail уведомление

## NPS (Net Promoter Score)

- Имеет подразделы для сбора и анализа обратной связи

## Акции

### Создание акции:

- Активность, заголовок, краткое/подробное описание, фото
- Привязка к филиалам, период действия
- Привязка товаров, промокод
- Кнопки-ссылки на ресурсы
- Сортировка (порядок отображения)
- Фильтрация по меткам клиентов
- Выбор платформ

## Истории

- Формат Stories (аналог Instagram) для маркетинговых целей

## Промокоды

### Создание промокода:

- Название, коды (уникальные), описание, статус
- **5 типов**: 1) Скидка %, 2) Скидка фикс, 3) Подарки, 4) Бесплатные товары, 5) Воронки
- **Условия**: многоразовый, одноразовый для клиента, не суммировать, первый заказ
- **Ограничения**: кол-во товаров, обязательные товары, сумма заказа, время, дни недели, способ получения, платформы, период, филиалы
- **Массовая генерация** промокодов по шаблону
- **Действия**: активировать/деактивировать, привязать/отвязать от филиала, копировать
- **Экспорт** промокодов

## Воронки

- Сложные цепочки условий для акций и автоматизации

## Рассылки

- Push-уведомления и Telegram-рассылки

## Программа лояльности

### Статусы (Золотой, Серебряный, Бронзовый...):

- Название
- Макс. скидка (% от заказа, 1 бонус = 1 руб)
- Мин. статусных баллов для получения статуса
- Статусные баллы — только копятся, не списываются

### Правила:

- Заголовок (только для CRM)
- Для статуса (привязка)
- Тип правила
- Кэшбек (% начисления за заказ)
- Срок жизни бонусных баллов (дни)
- Срок жизни статусных баллов (дни)

### Транзакции:

- Тип: зачисление / списание
- Телефон клиента
- Количество баллов
- Срок жизни баллов
- Комментарий

## Метки

- Маркировка клиентов и заказов для сегментации

## Сотрудники

- Управление сотрудниками (в разработке в документации)

## Отчёты

### 4 типа отчётов:

1. **Заказов** — фильтры: филиалы, тип, платформы, подарки, скидки, метки, UTM, дата
2. **Товаров** — фильтры: филиалы, дата
3. **Промокодов** — фильтры: филиалы, источники, дата, код
4. **Промокода** — детальный отчёт по конкретному промокоду

- Экспорт всех отчётов в файл
